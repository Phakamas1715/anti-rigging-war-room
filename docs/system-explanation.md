# ระบบตรวจจับการทุจริตการเลือกตั้ง (Anti-Rigging War Room)

## บทนำ

ระบบ Anti-Rigging War Room เป็นแพลตฟอร์มที่ออกแบบมาเพื่อตรวจจับความผิดปกติในการเลือกตั้ง โดยใช้หลักการทางสถิติและนิติวิทยาศาสตร์ระดับสากล ระบบนี้รวบรวมข้อมูลจากอาสาสมัครภาคสนาม นำมาเปรียบเทียบกับผลทางการ และวิเคราะห์หาความผิดปกติด้วยเครื่องมือทางสถิติ 5 ประเภท

---

## สถาปัตยกรรมระบบ

ระบบประกอบด้วย 3 ส่วนหลัก ได้แก่ การรวบรวมข้อมูล การวิเคราะห์ และการแจ้งเตือน โดยข้อมูลจะไหลจากอาสาสมัครภาคสนามผ่านแอปพลิเคชันมือถือ เข้าสู่ฐานข้อมูลกลาง แล้วถูกประมวลผลด้วยอัลกอริทึมวิเคราะห์แบบ Real-time

| ส่วนประกอบ | หน้าที่ | เทคโนโลยี |
|-----------|--------|----------|
| Frontend | แอปอาสาสมัคร, Dashboard | React, TypeScript, Tailwind CSS |
| Backend | API, การวิเคราะห์ | Node.js, tRPC, Express |
| Database | จัดเก็บข้อมูล | MySQL/TiDB |
| Storage | รูปภาพหลักฐาน | S3 |
| AI/ML | OCR อ่านตัวเลข | Gemini Vision, DeepSeek |

## วิธีการเพิ่มข้อมูลเข้าระบบ

### 1. ผ่านอาสาสมัครภาคสนาม (แนะนำ)

วิธีนี้เป็นวิธีหลักในการรวบรวมข้อมูล โดยอาสาสมัครจะประจำอยู่ที่หน่วยเลือกตั้งและส่งข้อมูลแบบ Real-time

**ขั้นตอนการทำงาน:**

1. **ลงทะเบียน** - อาสาสมัครเข้าหน้า `/volunteer/register` กรอกชื่อ เบอร์โทร และ LINE ID ระบบจะออกรหัส 6 หลักให้อัตโนมัติ

2. **เข้าสู่ระบบ** - ใช้รหัส 6 หลักที่ได้รับ Login ที่หน้า `/volunteer/login`

3. **ถ่ายรูปกระดานนับคะแนน** - เมื่อหน่วยเลือกตั้งนับคะแนนเสร็จ อาสาสมัครถ่ายรูปกระดานที่แสดงผลคะแนน

4. **ระบบ OCR อ่านตัวเลข** - ระบบใช้ AI (Gemini Vision) อ่านตัวเลขจากรูปภาพอัตโนมัติ

5. **ยืนยันและส่งข้อมูล** - อาสาสมัครตรวจสอบความถูกต้อง แก้ไขหากจำเป็น แล้วกดส่ง

6. **เปรียบเทียบอัตโนมัติ** - ระบบเปรียบเทียบกับผลทางการทันที หากพบส่วนต่างจะแจ้งเตือน

### 2. ผ่าน Admin Dashboard

สำหรับการนำเข้าข้อมูลจำนวนมากหรือข้อมูลย้อนหลัง

**เมนูที่เกี่ยวข้อง:**

| เมนู | หน้าที่ |
|-----|--------|
| นำเข้าข้อมูล | Import CSV/Excel ข้อมูลหน่วยเลือกตั้งและผลคะแนน |
| Batch OCR | อัปโหลดรูปภาพหลายรูปพร้อมกัน ระบบ OCR อ่านทั้งหมด |
| PVT Comparison | เพิ่ม/แก้ไขผลคะแนนด้วยตนเอง |

## เครื่องมือวิเคราะห์ทางสถิติ

### 1. Klimek Model (การตรวจจับการยัดบัตร/ขโมยคะแนน)

Klimek Model พัฒนาโดย Peter Klimek และทีมจาก Medical University of Vienna เป็นวิธีการทางสถิติที่ใช้ตรวจจับการทุจริตการเลือกตั้ง 2 รูปแบบหลัก [1]

**หลักการทำงาน:**

ระบบวิเคราะห์ความสัมพันธ์ระหว่าง **อัตราการออกมาใช้สิทธิ์ (Turnout)** กับ **สัดส่วนคะแนนของผู้ชนะ (Vote Share)** ในการเลือกตั้งที่โปร่งใส ทั้งสองตัวแปรนี้ไม่ควรมีความสัมพันธ์กันอย่างมีนัยสำคัญ

**ค่าที่วัด:**

| ค่า | ความหมาย | เกณฑ์ปกติ | สัญญาณผิดปกติ |
|----|---------|----------|--------------|
| Alpha (α) | Vote Stuffing - การยัดบัตรลงหีบ | < 0.05 | > 0.05 |
| Beta (β) | Vote Stealing - การขโมยคะแนนจากคู่แข่ง | < 0.03 | > 0.03 |

**การตีความ:**

เมื่อพบว่าหน่วยเลือกตั้งที่มี Turnout สูงผิดปกติ มักจะมีสัดส่วนคะแนนของผู้ชนะสูงตามไปด้วย นั่นอาจบ่งชี้ว่ามีการยัดบัตรปลอมเพิ่มเข้าไป ระบบจะแสดงผลเป็น Heatmap โดยหน่วยที่อยู่ในโซนสีแดง (มุมขวาบน) ถือว่าน่าสงสัย

### 2. Benford's Law (กฎของเบนฟอร์ด)

Benford's Law หรือ "กฎหลักแรก" เป็นปรากฏการณ์ทางสถิติที่พบว่าในชุดข้อมูลตัวเลขที่เกิดขึ้นตามธรรมชาติ ตัวเลขหลักแรกจะไม่กระจายตัวเท่ากัน แต่เลข 1 จะปรากฏบ่อยที่สุด (ประมาณ 30%) และลดลงตามลำดับ [2]

**การประยุกต์ใช้กับการเลือกตั้ง:**

ระบบใช้ **Second Digit Benford's Law (2BL)** วิเคราะห์ตัวเลขหลักที่สองของผลคะแนน เนื่องจากการปลอมแปลงตัวเลขด้วยมือมักจะสร้างรูปแบบที่ผิดปกติ

**ค่าที่วัด:**

| ค่า | ความหมาย | เกณฑ์ปกติ |
|----|---------|----------|
| Chi-Square | ค่าความแตกต่างจากการกระจายตัวตามทฤษฎี | < 15.51 (df=9, p=0.05) |
| P-Value | ความน่าจะเป็นที่ข้อมูลเป็นไปตามธรรมชาติ | > 0.05 |

**การตีความ:**

หาก Chi-Square สูงและ P-Value ต่ำ แสดงว่าการกระจายตัวของตัวเลขผิดปกติ อาจมีการแก้ไขตัวเลขด้วยมือ

### 3. Social Network Analysis (SNA) - การวิเคราะห์เครือข่าย

SNA ใช้ทฤษฎีกราฟวิเคราะห์ความสัมพันธ์ระหว่างบุคคลหรือหน่วยงานที่เกี่ยวข้องกับการเลือกตั้ง เพื่อค้นหา "Hub" หรือจุดศูนย์กลางที่อาจเป็นผู้ประสานงานการทุจริต [3]

**ค่าที่วัด:**

| ค่า | ความหมาย | สัญญาณผิดปกติ |
|----|---------|--------------|
| Centrality Score | ความเป็นศูนย์กลางของโหนด | > 0.7 |
| Betweenness | ความสำคัญในการเชื่อมต่อเครือข่าย | สูงผิดปกติ |
| Clustering Coefficient | ความหนาแน่นของกลุ่ม | สูงผิดปกติ |

**การตีความ:**

บุคคลที่มี Centrality Score สูงและเชื่อมต่อกับหลายหน่วยเลือกตั้งที่พบความผิดปกติ อาจเป็นผู้ประสานงานการทุจริต

### 4. Parallel Vote Tabulation (PVT) - การนับคะแนนคู่ขนาน

PVT เป็นวิธีการมาตรฐานสากลที่องค์กรสังเกตการณ์การเลือกตั้งใช้ทั่วโลก โดยส่งอาสาสมัครไปประจำหน่วยเลือกตั้ง บันทึกผลคะแนนจริง แล้วนำมาเปรียบเทียบกับผลทางการ [4]

**ค่าที่วัด:**

| ค่า | ความหมาย | เกณฑ์ปกติ |
|----|---------|----------|
| Gap | ส่วนต่างระหว่างคะแนนที่เรานับ vs ทางการ | < 1% |
| Coverage | สัดส่วนหน่วยที่มีอาสาสมัครครอบคลุม | > 80% |

**การตีความ:**

หากพบ Gap มากกว่า 1% ในหน่วยใดหน่วยหนึ่ง หรือพบรูปแบบ Gap ที่เอนเอียงไปทางผู้สมัครคนใดคนหนึ่งอย่างเป็นระบบ ถือว่าน่าสงสัย

### 5. Spatial Analysis (Z-Score) - การวิเคราะห์เชิงพื้นที่

Spatial Analysis ใช้หลักการทางภูมิศาสตร์วิเคราะห์ว่าหน่วยเลือกตั้งที่อยู่ใกล้กันควรมีพฤติกรรมการลงคะแนนคล้ายกัน หากพบหน่วยที่มีผลลัพธ์แตกต่างจากเพื่อนบ้านอย่างมาก อาจบ่งชี้ความผิดปกติ [5]

**ค่าที่วัด:**

| ค่า | ความหมาย | เกณฑ์ปกติ |
|----|---------|----------|
| Z-Score | ค่าเบี่ยงเบนมาตรฐานจากค่าเฉลี่ยเพื่อนบ้าน | -2 ถึง +2 |
| Moran's I | ค่าความสัมพันธ์เชิงพื้นที่ | ใกล้ 0 |

**การตีความ:**

หน่วยที่มี Z-Score > 2 หรือ < -2 ถือว่าผิดปกติจากบริบทพื้นที่ ควรตรวจสอบเพิ่มเติม

## ระบบแจ้งเตือน

ระบบสามารถส่งการแจ้งเตือนผ่าน 2 ช่องทาง เมื่อตรวจพบความผิดปกติ

| ช่องทาง | ประเภทการแจ้งเตือน |
|--------|-------------------|
| Discord Webhook | Rich Embed พร้อมสีตามระดับความรุนแรง |
| LINE Notify | ข้อความพร้อมรายละเอียดหน่วยที่พบปัญหา |

**ประเภทการแจ้งเตือน:**

1. **Klimek Alert** - เมื่อพบ Alpha > 0.05 หรือ Beta > 0.03
2. **PVT Gap Alert** - เมื่อพบส่วนต่างคะแนนเกินเกณฑ์
3. **Benford Alert** - เมื่อ P-Value < 0.05
4. **Network Hub Alert** - เมื่อพบบุคคลที่มี Centrality Score > 0.7
5. **Spatial Alert** - เมื่อพบหน่วยที่มี Z-Score > 2
